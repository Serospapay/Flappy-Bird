# Етапи розробки проєкту: Flappy Bird - Enhanced Edition

Документ описує всі етапи розробки гри з шматками коду та детальними технічними описами. Нічого не опущено.

---

## Технологічний стек

- **Python 3.14+**
- **pygame-ce 2.5.6+** (SDL 2.32)
- **numpy** (опційно, для генерації звуків)

**Структура:** `src/` (ядро), `systems/` (системи), `effects/` (ефекти), `ui/` (інтерфейс), `utils/` (утиліти).

**Архітектура:** State Pattern (`menu`, `playing`, `paused`, `game_over`, `settings`, `achievements`, `statistics`, `skins`), модульний підхід, збереження JSON.

**Керування:** SPACE — стрибок; DOWN — dash; ESC — пауза; LMB — клік.

## Етап 1. Інфраструктура та конфігурація

### Опис і призначення

Централізація всіх констант у одному місці для простоти налаштування гри. Один клас `GameConfig` змінюється при зміні фізики, розмірів екрану, параметрів труб.

### Технічне завдання

- Визначити розміри екрану, FPS, колір фону
- Параметри фізики птаха: гравітація, сила стрибка, dash
- Параметри труб: ширина, проміжок, швидкість, інтервал spawn
- Межі для динамічної складності (мін. проміжок, макс. швидкість)
- Скрипти запуску (run.bat) та встановлення (install.bat) з перевіркою залежностей

### Реалізація

```python
# src/game_config.py
SCREEN_WIDTH = 800
SCREEN_HEIGHT = 600
FPS = 60
GRAVITY = 0.6
JUMP_STRENGTH = -10
DASH_SPEED = -18
DASH_COOLDOWN = 120
PIPE_WIDTH = 80
PIPE_GAP = 200
PIPE_SPAWN_INTERVAL = 1800
MIN_PIPE_GAP = 150
```

```batch
python utils\check_dependencies.py
python src\game.py
```

### Пояснення

- Усі числові значення в одному модулі; зміна FPS, розмірів або фізики не потребує пошуку по всьому проєкту
- Кольори `BIRD_COLOR`, `PIPE_COLOR` використовуються як fallback, якщо немає скінів
- `PIPE_SPAWN_INTERVAL` в мілісекундах; `DASH_COOLDOWN` та `GHOST_DURATION` — у кадрах

## Етап 2. Ядро гри — Bird та Pipe

### Опис і призначення

Bird — гравець з фізикою, стрибком, dash і ghost. Pipe — перешкоди з різними типами. Pipe.draw — pseudo-3D і шапка для візуальної деталізації.

### Технічне завдання

- **Bird:** гравітація, обмеження швидкості, ротація (-90°…+30°), dash (DOWN, cooldown 2 сек), ghost (power-up)

```python
# Bird.update — гравітація, ротація velocity*3
self.rotation = min(30, max(-90, self.velocity * 3))
```

```python
# src/bird.py — тіло еліпсом

```

```python
# src/pipe.py — типи + pseudo-3D
TYPE_NORMAL, TYPE_MOVING, TYPE_SHRINKING, TYPE_SPIKES, TYPE_SPLIT
```

### Пояснення

## Етап 3. PipeManager — генерація, колізії, рахунок

### Опис і призначення

Менеджер труб відповідає за спавн з урахуванням інтервалу, тип труб від рахунку, RNG для Daily Challenge, перевірку колізій і нарахування очок.

### Технічне завдання

- Спавн з інтервалом `PIPE_SPAWN_INTERVAL` (мс)
- Типи труб: score 0–10 — normal; 10+ — moving; 15+ — spikes; 20+ — shrinking; 30+ — split
- Для Daily Challenge — `random.Random(seed)` з seed = дата
- `check_collision(bird, ghost_active)` — колізія з `pipe.get_rects()`; ghost ігнорує
- Рахунок: труба вважається пройденою, коли `pipe.x + pipe.width < bird_x`

### Реалізація

```python
# PipeManager — spawn, types_pool залежить від score
types_pool = [Pipe.TYPE_NORMAL]
if score > 10: types_pool.extend([Pipe.TYPE_MOVING])
if score > 30: types_pool.extend([Pipe.TYPE_SPLIT])
pipe_type = self._rng.choice(types_pool)
```

```python
# check_collision — ghost ігнорує
if bird is None or ghost_active: return False
for rect in pipe.get_rects():
    if bird_rect.colliderect(rect): return True
```

```python
# check_score — pipe.passed уникає подвійного рахунку
if not pipe.passed and pipe.x + pipe.width < bird_x:
    pipe.passed = True
    self.score += 1
```

### Пояснення

- `random_gen` дозволяє передати `GameModes.get_daily_random()` для однакової щоденної генерації
- `types_pool` розширюється з ростом рахунку — складніші труби з’являються пізніше
- `pipe.passed` уникає подвійного зарахування очок

## Етап 4. Power-up система

### Опис і призначення

Бонуси: shield (один удар), slow_time (0.5x швидкість), double_score, coin (монета), ghost (прохід крізь труби). Малювання як пульсуючі еліпси; shield/ghost — напівпрозоре коло навколо птаха.

### Технічне завдання

- 5 типів power-up з різними ефектами і тривалістю
- Активація при зборі; shield витрачається при першому зіткненні
- slow_time змінює фактичну швидкість (pipe_speed * 0.5)
- Візуал: пульсуючий еліпс з символом (S, T, x2, C, G)
- Щит/ghost: Surface SRCALPHA, колір (100, 200, 255, 120), радіус птах + 18 px

### Реалізація

```python
# PowerUp.draw — пульсація, еліпс
pulse = int(5 * abs(math.sin(self.animation_time * 0.2)))
size = self.width + pulse
pygame.draw.ellipse(screen, color, rect)
```

```python
# PowerUpManager — active_effects, speed_multiplier
'shield': {'active': False, 'duration': 600}
'slow_time': {'active': False, 'duration': 300}
def get_speed_multiplier(self):
    return 0.5 if self.is_slow_time_active() else 1.0
```

### Пояснення

- `shield`: тривалість 600 кадрів (10 сек), але знімається при першому зіткненні через `use_shield()`
- `ghost`: 90 кадрів (1.5 сек), колізії з трубами не застосовуються
- `coin` не дає ефекту — лише збір монет; більшість power-up спавняться в проміжках між трубами

## Етап 7. Звук та музика

### Опис і призначення

Програмна генерація звуків без зовнішніх файлів. jump, coin, collision, powerup, achievement. Опційна динамічна музика за mood (calm/tension). Окрема гучність для музики та SFX.

### Технічне завдання

- pygame.mixer.init(frequency=22050, size=-16, channels=2)
- Генерація через sine-хвилі (numpy або math.sin)
- Обмеження wave до int16 (-32767…32767)
- set_music_volume, set_sfx_volume з збереженням в settings

### Реалізація

```python
# SoundManager — програмна генерація
pygame.mixer.init(frequency=22050, size=-16, channels=2)
# jump, coin, collision, powerup, achievement — sine-хвилі
wave = 4096 * np.sin(2.0 * np.pi * frequency * t)
wave = np.clip(wave, -32767, 32767).astype(np.int16)
```

### Пояснення

- 4096 — амплітуда; np.clip гарантує валідний int16
- jump — послідовність висхідних тонів (440, 550, 660 Hz)
- collision — низький тон; coin — короткий високий

## Етап 8. Динамічна складність

### Опис і призначення

Progressive difficulty: з ростом рахунку зменшується проміжок, збільшується швидкість, зменшується інтервал spawn. Застосовується до PipeManager і config.

### Технічне завдання

- difficulty_level = score // 10
- gap = max(MIN, base_gap - level*10)
- speed = min(8.0, base_speed + level*0.5)
- spawn_interval = max(1200, base - level*100)
- Оновлювати існуючі труби (крім SHRINKING)

### Реалізація

```python
# get_difficulty_params — level = score // 10
gap = max(MIN, base_gap - level * 10)
speed = min(8.0, base_speed + level * 0.5)
spawn_interval = max(1200, base - level * 100)
```

```python
# apply_difficulty — оновлюємо config та існуючі труби
for pipe in pipe_manager.pipes:
    if pipe.type != Pipe.TYPE_SHRINKING:
        pipe.gap_size = params['gap']
    pipe.config.PIPE_SPEED = params['speed']
```

### Пояснення

- SHRINKING має власну логіку зменшення gap, її не перезаписуємо
- Застосування кожні 10 очок робить гру поступово складнішою без різких стрибків

```python
return random.Random(seed or GameModes.get_daily_seed())
```


- get_panel_surface — повернути з кешу або створити і закешувати
- Очищення lists замість створення нових (particles, animations)

### Реалізація

```python
# CacheManager — ключ (width, height, bg_color, alpha, border_radius)
cache_key = (width, height, bg_color, alpha, ...)
if cache_key in self.panel_cache:
    return self.panel_cache[cache_key]
```

### Пояснення

- Панелі однакового розміру/кольору малюються один раз
- max_cache_size запобігає необмеженому зростанню пам'яті

## Етап 16. Slider та Toast (деталі)

### Опис і призначення

Slider — track, fill, handle; callback при зміні; label по centerx. Toast — message, type, duration; fade in/out; колір за типом.

### Технічне завдання

- Slider: set_value_from_pos(x) за позицією миші; get_ratio()
- Toast: update() зменшує duration; alpha та y_offset для fade
- ToastSystem керує списком toast, draw по черзі

### Реалізація

```python
# Slider — set_value_from_pos за x миші, callback(value)
ratio = (x - track_start) / track_width
self.value = self.min_value + ratio * (self.max_value - self.min_value)
self.callback(self.value)
```

```python
# Toast.update — fade in перші 15 кадрів, fade out останні 15
self.duration -= 1
return self.duration > 0
```

### Пояснення

- Slider callback викликається при drag і змінює settings
- Toast fade in/out — перші та останні 15 кадрів
